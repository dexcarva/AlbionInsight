## مدونة قواعد السلوك

يخضع هذا المشروع وكل من يشارك فيه لمدونة قواعد السلوك الخاصة بنا. من خلال المشاركة، يُتوقع منك الالتزام بهذه المدونة. يرجى الإبلاغ عن السلوك غير المقبول لمشرفي المشروع.

## كيف يمكنني المساهمة؟

### الإبلاغ عن الأخطاء

قبل إنشاء تقارير الأخطاء، يرجى التحقق من المشاكل الموجودة لتجنب التكرار. عند إنشاء تقرير خطأ، قم بتضمين أكبر قدر ممكن من التفاصيل باستخدام قالب تقرير الأخطاء.

**تقارير الأخطاء الجيدة تتضمن:**
- عنوان واضح ووصفي
- خطوات دقيقة لإعادة إنتاج المشكلة
- السلوك المتوقع مقابل السلوك الفعلي
- لقطات شاشة إذا كانت قابلة للتطبيق
- تفاصيل بيئتك (نظام التشغيل، إصدار بايثون، إلخ)
- سجلات أو رسائل خطأ ذات صلة

### اقتراح الميزات

نرحب باقتراحات الميزات! يرجى استخدام قالب طلب الميزة وتقديم:
- وصف واضح للميزة
- المشكلة التي تحلها
- مقاربات التنفيذ الممكنة
- أي بدائل فكرت فيها

### المساهمات البرمجية

نحن نحب المساهمات البرمجية! إليك كيفية البدء:

1. **تفرع المستودع (Fork)** وأنشئ فرعك من `master`
2. **قم بإعداد بيئة التطوير الخاصة بك** (انظر إعداد بيئة التطوير أدناه)
3. **قم بإجراء تغييراتك** باتباع معايير الترميز الخاصة بنا
4. **اختبر تغييراتك** بدقة
5. **حدث التوثيق** إذا لزم الأمر
6. **أرسل طلب سحب (Pull Request)** باستخدام قالب طلب السحب الخاص بنا

### التوثيق

التحسينات على التوثيق محل تقدير دائمًا! ويشمل ذلك:
- ملفات README
- صفحات Wiki
- تعليقات الكود
- البرامج التعليمية والأدلة
- الترجمات إلى لغات أخرى

## إعداد بيئة التطوير

### المتطلبات الأساسية

- بايثون 3.8 أو أعلى
- Git
- امتيازات الجذر/المسؤول (لالتقاط الحزم)

### إعداد بيئتك

```bash
# استنساخ تفرعك
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# إنشاء بيئة افتراضية
python3 -m venv venv

# تفعيل البيئة الافتراضية
# على Linux/macOS:
source venv/bin/activate
# على Windows:
venv\Scripts\activate

# تثبيت التبعيات
pip install -r requirements.txt

# تثبيت تبعيات التطوير
pip install pylint flake8 black pytest
```

### تشغيل التطبيق

```bash
# على Linux/macOS:
sudo venv/bin/python3 albion_insight.py

# على Windows (كمسؤول):
python albion_insight.py
```

## معايير الترميز

نحن نتبع إرشادات نمط PEP 8 لكود بايثون. يرجى التأكد من أن الكود الخاص بك يلتزم بهذه المعايير:

- استخدم 4 مسافات للمسافة البادئة (بدون علامات جدولة)
- الحد الأقصى لطول السطر هو 100 حرف
- استخدم أسماء متغيرة ووظيفة ذات مغزى
- أضف docstrings لجميع الوظائف والفئات
- قم بتضمين تلميحات النوع (type hints) عند الاقتضاء
- حافظ على تركيز ووظائف موجزة

**أدوات للمساعدة:**
```bash
# تنسيق الكود الخاص بك باستخدام black
black albion_insight.py

# التحقق من مشاكل النمط
flake8 albion_insight.py

# تشغيل المدقق (linter)
pylint albion_insight.py
```

## رسائل الالتزام (Commit Messages)

اكتب رسائل التزام واضحة وذات مغزى:

- استخدم صيغة المضارع ("Add feature" وليس "Added feature")
- استخدم صيغة الأمر ("Move cursor to..." وليس "Moves cursor to...")
- حدد السطر الأول بـ 72 حرفًا
- أشر إلى المشاكل وطلبات السحب عند الاقتضاء

**أمثلة:**
```
Add damage meter export functionality

Fix network packet parsing for IPv6 connections

Update README with macOS installation instructions

Closes #123
```

## عملية طلب السحب (Pull Request Process)

1. **حدث التوثيق** لأي تغييرات في الوظائف
2. **أضف اختبارات** للميزات الجديدة أو إصلاحات الأخطاء
3. **تأكد من اجتياز جميع الاختبارات** قبل الإرسال
4. **حدث ملف README.md** إذا لزم الأمر
5. **املأ قالب طلب السحب** بالكامل
6. **اربط المشاكل ذات الصلة** في وصف طلب السحب الخاص بك
7. **اطلب مراجعة** من المشرفين
8. **تعامل مع الملاحظات** على الفور وباحترافية

### قائمة مراجعة طلب السحب (PR Checklist)

قبل إرسال طلب السحب الخاص بك، تأكد مما يلي:
- [ ] الكود يتبع إرشادات نمط المشروع
- [ ] تم الانتهاء من المراجعة الذاتية
- [ ] تمت إضافة تعليقات إلى أقسام الكود المعقدة
- [ ] تم تحديث التوثيق
- [ ] لم يتم إنشاء تحذيرات جديدة
- [ ] تمت إضافة الاختبارات واجتيازها
- [ ] تم دمج التغييرات التابعة
- [ ] تم ملء قالب طلب السحب بالكامل
