## کد رفتار

این پروژه و همه شرکت‌کنندگان در آن تحت حاکمیت کد رفتار ما هستند. انتظار می‌رود که با شرکت در آن، این کد را رعایت کنید. لطفاً رفتار غیرقابل قبول را به نگهدارندگان پروژه گزارش دهید.

## چگونه می‌توانم مشارکت کنم؟

### گزارش اشکالات

قبل از ایجاد گزارش‌های اشکال، لطفاً مسائل موجود را بررسی کنید تا از تکرار جلوگیری شود. هنگامی که یک گزارش اشکال ایجاد می‌کنید، تا حد امکان جزئیات بیشتری را با استفاده از الگوی گزارش اشکال وارد کنید.

**گزارش‌های اشکال خوب شامل موارد زیر است:**
- یک عنوان واضح و توصیفی
- مراحل دقیق برای بازتولید مشکل
- رفتار مورد انتظار در مقابل رفتار واقعی
- تصاویر صفحه در صورت لزوم
- جزئیات محیط شما (سیستم عامل، نسخه پایتون و غیره)
- گزارش‌های مربوطه یا پیام‌های خطا

### پیشنهاد ویژگی‌ها

پیشنهادات ویژگی‌ها مورد استقبال قرار می‌گیرند! لطفاً از الگوی درخواست ویژگی استفاده کنید و موارد زیر را ارائه دهید:
- شرح واضحی از ویژگی
- مشکلی که حل می‌کند
- رویکردهای احتمالی پیاده‌سازی
- هرگونه جایگزینی که در نظر گرفته‌اید

### مشارکت‌های کد

ما مشارکت‌های کد را دوست داریم! در اینجا نحوه شروع آمده است:

1. **مخزن را فورک کنید** و شاخه خود را از `master` ایجاد کنید
2. **محیط توسعه خود را تنظیم کنید** (به تنظیمات توسعه در زیر مراجعه کنید)
3. **تغییرات خود را اعمال کنید** با رعایت استانداردهای کدنویسی ما
4. **تغییرات خود را به طور کامل آزمایش کنید**
5. **مستندات را به‌روزرسانی کنید** در صورت نیاز
6. **یک pull request ارسال کنید** با استفاده از الگوی PR ما

### مستندسازی

بهبود مستندات همیشه مورد استقبال قرار می‌گیرد! این شامل موارد زیر است:
- فایل‌های README
- صفحات ویکی
- نظرات کد
- آموزش‌ها و راهنماها
- ترجمه به زبان‌های دیگر

## تنظیمات توسعه

### پیش‌نیازها

- پایتون 3.8 یا بالاتر
- Git
- امتیازات روت/مدیر (برای ضبط بسته)

### تنظیم محیط شما

```bash
# فورک خود را کلون کنید
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# یک محیط مجازی ایجاد کنید
python3 -m venv venv

# محیط مجازی را فعال کنید
# در لینوکس/macOS:
source venv/bin/activate
# در ویندوز:
venv\Scripts\activate

# وابستگی‌ها را نصب کنید
pip install -r requirements.txt

# وابستگی‌های توسعه را نصب کنید
pip install pylint flake8 black pytest
```

### اجرای برنامه

```bash
# در لینوکس/macOS:
sudo venv/bin/python3 albion_insight.py

# در ویندوز (به عنوان مدیر):
python albion_insight.py
```

## استانداردهای کدنویسی

ما از دستورالعمل‌های سبک PEP 8 برای کد پایتون پیروی می‌کنیم. لطفاً اطمینان حاصل کنید که کد شما به این استانداردها پایبند است:

- از 4 فاصله برای تورفتگی استفاده کنید (بدون تب)
- حداکثر طول خط 100 کاراکتر
- از نام‌های معنی‌دار برای متغیرها و توابع استفاده کنید
- docstringها را به همه توابع و کلاس‌ها اضافه کنید
- از راهنمایی‌های نوع (type hints) در صورت لزوم استفاده کنید
- توابع را متمرکز و مختصر نگه دارید

**ابزارهایی برای کمک:**
```bash
# کد خود را با black فرمت کنید
black albion_insight.py

# مشکلات سبک را بررسی کنید
flake8 albion_insight.py

# linter را اجرا کنید
pylint albion_insight.py
```

## پیام‌های کامیت

پیام‌های کامیت واضح و معنی‌دار بنویسید:

- از زمان حال استفاده کنید ("افزودن ویژگی" نه "ویژگی اضافه شد")
- از حالت امری استفاده کنید ("انتقال مکان‌نما به..." نه "مکان‌نما را به... منتقل می‌کند")
- خط اول را به 72 کاراکتر محدود کنید
- به مسائل و pull requestها در صورت لزوم ارجاع دهید

**مثال‌ها:**
```
افزودن قابلیت خروجی گرفتن از متر آسیب
	
رفع تجزیه بسته شبکه برای اتصالات IPv6
	
به‌روزرسانی README با دستورالعمل‌های نصب macOS
	
بستن #123
```

## فرآیند Pull Request

1. **مستندات را به‌روزرسانی کنید** برای هرگونه تغییر در عملکرد
2. **تست‌ها را اضافه کنید** برای ویژگی‌های جدید یا رفع اشکالات
3. **اطمینان حاصل کنید که همه تست‌ها با موفقیت اجرا می‌شوند** قبل از ارسال
4. **README.md را به‌روزرسانی کنید** در صورت نیاز
5. **الگوی PR را به طور کامل پر کنید**
6. **مسائل مرتبط را پیوند دهید** در توضیحات PR خود
7. **درخواست بررسی کنید** از نگهدارندگان
8. **بازخورد را به سرعت و حرفه‌ای برطرف کنید**

### چک لیست PR

قبل از ارسال PR خود، اطمینان حاصل کنید که:
- [ ] کد از دستورالعمل‌های سبک پروژه پیروی می‌کند
- [ ] خود-بررسی تکمیل شده است
- [ ] نظرات به بخش‌های کد پیچیده اضافه شده است
- [ ] مستندات به‌روزرسانی شده است
- [ ] هیچ هشدار جدیدی تولید نشده است
- [ ] تست‌ها اضافه شده و با موفقیت اجرا می‌شوند
- [ ] تغییرات وابسته ادغام شده‌اند

## سوالات؟

- با نگهدارندگان تماس بگیرید
