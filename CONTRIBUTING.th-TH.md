## หลักจรรยาบรรณ

โปรเจกต์นี้และทุกคนที่เข้าร่วมอยู่ภายใต้หลักจรรยาบรรณของเรา โดยการเข้าร่วม คุณคาดว่าจะยึดถือหลักจรรยาบรรณนี้ โปรดรายงานพฤติกรรมที่ไม่เหมาะสมต่อผู้ดูแลโปรเจกต์

## ฉันจะสามารถมีส่วนร่วมได้อย่างไร?

### การรายงานข้อบกพร่อง

ก่อนที่จะสร้างรายงานข้อบกพร่อง โปรดตรวจสอบ Issues ที่มีอยู่เพื่อหลีกเลี่ยงการซ้ำซ้อน เมื่อคุณสร้างรายงานข้อบกพร่อง โปรดใส่รายละเอียดให้มากที่สุดโดยใช้เทมเพลตรายงานข้อบกพร่อง

**รายงานข้อบกพร่องที่ดีควรมี:**
- ชื่อที่ชัดเจนและสื่อความหมาย
- ขั้นตอนที่แน่นอนในการทำซ้ำปัญหา
- พฤติกรรมที่คาดหวังเทียบกับพฤติกรรมจริง
- ภาพหน้าจอหากมี
- รายละเอียดสภาพแวดล้อมของคุณ (OS, เวอร์ชัน Python, ฯลฯ)
- บันทึกหรือข้อความแสดงข้อผิดพลาดที่เกี่ยวข้อง

### การแนะนำคุณสมบัติ

ยินดีรับข้อเสนอแนะคุณสมบัติ! โปรดใช้เทมเพลตคำขอคุณสมบัติและระบุ:
- คำอธิบายที่ชัดเจนของฟีเจอร์
- ปัญหาที่ฟีเจอร์นั้นแก้ไข
- แนวทางการนำไปใช้ที่เป็นไปได้
- ทางเลือกอื่นที่คุณพิจารณา

### การมีส่วนร่วมในโค้ด

เรายินดีรับการมีส่วนร่วมในโค้ด! นี่คือวิธีการเริ่มต้น:

1. **Fork ที่เก็บ** และสร้างสาขาของคุณจาก `master`
2. **ตั้งค่าสภาพแวดล้อมการพัฒนาของคุณ** (ดู การตั้งค่าการพัฒนา ด้านล่าง)
3. **ทำการเปลี่ยนแปลงของคุณ** ตามมาตรฐานการเขียนโค้ดของเรา
4. **ทดสอบการเปลี่ยนแปลงของคุณ** อย่างละเอียด
5. **อัปเดตเอกสารประกอบ** หากจำเป็น
6. **ส่ง Pull Request** โดยใช้เทมเพลต PR ของเรา

### เอกสารประกอบ

การปรับปรุงเอกสารประกอบเป็นสิ่งที่น่ายินดีเสมอ! ซึ่งรวมถึง:
- ไฟล์ README
- หน้า Wiki
- ข้อคิดเห็นในโค้ด
- บทช่วยสอนและคู่มือ
- การแปลเป็นภาษาอื่น

## การตั้งค่าการพัฒนา

### ข้อกำหนดเบื้องต้น

- Python 3.8 หรือสูงกว่า
- Git
- สิทธิ์ Root/Administrator (สำหรับการดักจับแพ็กเก็ต)

### การตั้งค่าสภาพแวดล้อมของคุณ

```bash
# โคลน fork ของคุณ
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# สร้างสภาพแวดล้อมเสมือน
python3 -m venv venv

# เปิดใช้งานสภาพแวดล้อมเสมือน
# บน Linux/macOS:
source venv/bin/activate
# บน Windows:
venv\Scripts\activate

# ติดตั้งการพึ่งพา
pip install -r requirements.txt

# ติดตั้งการพึ่งพาสำหรับการพัฒนา
pip install pylint flake8 black pytest
```

### การเรียกใช้แอปพลิเคชัน

```bash
# บน Linux/macOS:
sudo venv/bin/python3 albion_insight.py

# บน Windows (ในฐานะ Administrator):
python albion_insight.py
```

## มาตรฐานการเขียนโค้ด

เราปฏิบัติตามแนวทางสไตล์ PEP 8 สำหรับโค้ด Python โปรดตรวจสอบให้แน่ใจว่าโค้ดของคุณเป็นไปตามมาตรฐานเหล่านี้:

- ใช้ 4 ช่องว่างสำหรับการเยื้อง (ไม่มีแท็บ)
- ความยาวบรรทัดสูงสุด 100 ตัวอักษร
- ใช้ชื่อตัวแปรและฟังก์ชันที่มีความหมาย
- เพิ่ม docstrings ให้กับฟังก์ชันและคลาสทั้งหมด
- รวม type hints เมื่อเหมาะสม
- รักษาฟังก์ชันให้เน้นและกระชับ

**เครื่องมือช่วยเหลือ:**
```bash
# จัดรูปแบบโค้ดของคุณด้วย black
black albion_insight.py

# ตรวจสอบปัญหาด้านสไตล์
flake8 albion_insight.py

# เรียกใช้ linter
pylint albion_insight.py
```

## ข้อความคอมมิต

เขียนข้อความคอมมิตที่ชัดเจนและมีความหมาย:

- ใช้กาลปัจจุบัน ("Add feature" ไม่ใช่ "Added feature")
- ใช้รูปคำสั่ง ("Move cursor to..." ไม่ใช่ "Moves cursor to...")
- จำกัดบรรทัดแรกไว้ที่ 72 ตัวอักษร
- อ้างอิง Issues และ Pull Requests เมื่อเกี่ยวข้อง

**ตัวอย่าง:**
```
Add damage meter export functionality

Fix network packet parsing for IPv6 connections

Update README with macOS installation instructions

Closes #123
```

## กระบวนการ Pull Request

1. **อัปเดตเอกสารประกอบ** สำหรับการเปลี่ยนแปลงใดๆ ต่อฟังก์ชันการทำงาน
2. **เพิ่มการทดสอบ** สำหรับคุณสมบัติใหม่หรือการแก้ไขข้อบกพร่อง
3. **ตรวจสอบให้แน่ใจว่าการทดสอบทั้งหมดผ่าน** ก่อนส่ง
4. **อัปเดต README.md** หากจำเป็น
5. **กรอกเทมเพลต PR** ให้สมบูรณ์
6. **เชื่อมโยง Issues ที่เกี่ยวข้อง** ในคำอธิบาย PR ของคุณ
7. **ขอการตรวจสอบ** จากผู้ดูแล
8. **จัดการข้อเสนอแนะ** อย่างรวดเร็วและเป็นมืออาชีพ

### รายการตรวจสอบ PR

ก่อนส่ง PR ของคุณ ตรวจสอบให้แน่ใจว่า:
- [ ] โค้ดเป็นไปตามแนวทางสไตล์ของโปรเจกต์
- [ ] การตรวจสอบตนเองเสร็จสมบูรณ์
- [ ] เพิ่มข้อคิดเห็นในส่วนโค้ดที่ซับซ้อน
- [ ] เอกสารประกอบได้รับการอัปเดต
- [ ] ไม่มีคำเตือนใหม่เกิดขึ้น
- [ ] การทดสอบถูกเพิ่มและผ่าน
- [ ] การเปลี่ยนแปลงที่ขึ้นต่อกันถูกรวมเข้าด้วยกัน
