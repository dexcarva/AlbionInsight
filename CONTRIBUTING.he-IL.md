## קוד התנהגות

פרויקט זה וכל מי שמשתתף בו כפופים לקוד ההתנהגות שלנו. על ידי השתתפות, מצופה מכם לקיים קוד זה. אנא דווחו על התנהגות בלתי הולמת למתחזקי הפרויקט.

## כיצד אוכל לתרום?

### דיווח על באגים

לפני יצירת דוחות באגים, אנא בדקו את הבעיות הקיימות כדי למנוע כפילויות. כאשר אתם יוצרים דוח באג, כללו כמה שיותר פרטים באמצעות תבנית דוח הבאג.

**דוחות באגים טובים כוללים:**
- כותרת ברורה ותיאורית
- שלבים מדויקים לשחזור הבעיה
- התנהגות צפויה לעומת התנהגות בפועל
- צילומי מסך אם רלוונטי
- פרטי הסביבה שלכם (מערכת הפעלה, גרסת Python וכו')
- יומנים רלוונטיים או הודעות שגיאה

### הצעת תכונות

הצעות לתכונות יתקבלו בברכה! אנא השתמשו בתבנית בקשת התכונה וספקו:
- תיאור ברור של התכונה
- הבעיה שהיא פותרת
- גישות יישום אפשריות
- כל חלופה ששקלתם

### תרומות קוד

אנחנו אוהבים תרומות קוד! כך מתחילים:

1. **בצעו Fork למאגר** וצרו את הענף שלכם מ-`master`
2. **הגדירו את סביבת הפיתוח שלכם** (ראו הגדרת פיתוח למטה)
3. **בצעו את השינויים שלכם** בהתאם לתקני הקידוד שלנו
4. **בדקו את השינויים שלכם** ביסודיות
5. **עדכנו את התיעוד** אם יש צורך
6. **שלחו בקשת משיכה** באמצעות תבנית ה-PR שלנו

### תיעוד

שיפורים בתיעוד תמיד מוערכים! זה כולל:
- קבצי README
- דפי Wiki
- הערות קוד
- מדריכים והנחיות
- תרגומים לשפות אחרות

## הגדרת פיתוח

### דרישות קדם

- Python 3.8 ומעלה
- Git
- הרשאות שורש/מנהל (ללכידת חבילות)

### הגדרת הסביבה שלכם

```bash
# שכפלו את ה-fork שלכם
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# צרו סביבה וירטואלית
python3 -m venv venv

# הפעילו את הסביבה הווירטואלית
# בלינוקס/macOS:
source venv/bin/activate
# בווינדוס:
venv\Scripts\activate

# התקינו תלויות
pip install -r requirements.txt

# התקינו תלויות פיתוח
pip install pylint flake8 black pytest
```

### הפעלת היישום

```bash
# בלינוקס/macOS:
sudo venv/bin/python3 albion_insight.py

# בווינדוס (כמנהל):
python albion_insight.py
```

## תקני קידוד

אנו עוקבים אחר הנחיות הסגנון של PEP 8 עבור קוד Python. אנא ודאו שהקוד שלכם עומד בתקנים אלה:

- השתמשו ב-4 רווחים להזחה (לא טאבים)
- אורך שורה מקסימלי של 100 תווים
- השתמשו בשמות משמעותיים למשתנים ופונקציות
- הוסיפו docstrings לכל הפונקציות והמחלקות
- כללו רמזי טיפוס (type hints) היכן שמתאים
- שמרו על פונקציות ממוקדות ותמציתיות

**כלים לעזרה:**
```bash
# פרמטו את הקוד שלכם עם black
black albion_insight.py

# בדקו בעיות סגנון
flake8 albion_insight.py

# הפעילו את ה-linter
pylint albion_insight.py
```

## הודעות קומיט

כתבו הודעות קומיט ברורות ומשמעותיות:

- השתמשו בזמן הווה ("הוסף תכונה" ולא "הוסיף תכונה")
- השתמשו בציווי ("הזז סמן ל..." ולא "מזיז סמן ל...")
- הגבילו את השורה הראשונה ל-72 תווים
- הפנו לבעיות ובקשות משיכה כאשר רלוונטי

**דוגמאות:**
```
הוסף פונקציונליות ייצוא של מד נזק
	
תקן ניתוח חבילות רשת עבור חיבורי IPv6
	
עדכן README עם הוראות התקנה עבור macOS
	
סוגר #123
```

## תהליך בקשת משיכה (Pull Request)

1. **עדכנו תיעוד** עבור כל שינוי בפונקציונליות
2. **הוסיפו בדיקות** עבור תכונות חדשות או תיקוני באגים
3. **ודאו שכל הבדיקות עוברות** לפני השליחה
4. **עדכנו את README.md** אם יש צורך
5. **מלאו את תבנית ה-PR** במלואה
6. **קשרו בעיות קשורות** בתיאור ה-PR שלכם
7. **בקשו סקירה** מהמתחזקים
8. **טפלו במשוב** במהירות ובמקצועיות

### רשימת בדיקה ל-PR

לפני שליחת ה-PR שלכם, ודאו ש:
- [ ] הקוד עומד בהנחיות הסגנון של הפרויקט
- [ ] בוצעה בדיקה עצמית
- [ ] הערות נוספו למקטעי קוד מורכבים
- [ ] התיעוד עודכן
- [ ] לא נוצרו אזהרות חדשות
- [ ] בדיקות נוספו ועוברות
- [ ] שינויים תלויים מוזגו
