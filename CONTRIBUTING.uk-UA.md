## Кодекс поведінки

Цей проєкт і всі, хто бере в ньому участь, керуються нашим Кодексом поведінки. Очікується, що беручи участь, ви будете дотримуватися цього кодексу. Будь ласка, повідомляйте про неприйнятну поведінку супровідникам проєкту.

## Як я можу зробити внесок?

### Повідомлення про помилки

Перш ніж створювати звіти про помилки, будь ласка, перевірте наявні проблеми, щоб уникнути дублікатів. Коли ви створюєте звіт про помилку, включіть якомога більше деталей, використовуючи шаблон звіту про помилку.

**Хороші звіти про помилки включають:**
- Чіткий і описовий заголовок
- Точні кроки для відтворення проблеми
- Очікувана та фактична поведінка
- Знімки екрана, якщо застосовно
- Деталі вашого середовища (ОС, версія Python тощо)
- Відповідні журнали або повідомлення про помилки

### Пропозиції функцій

Пропозиції функцій вітаються! Будь ласка, використовуйте шаблон запиту функції та надайте:
- Чіткий опис функції
- Проблема, яку вона вирішує
- Можливі підходи до реалізації
- Будь-які альтернативи, які ви розглядали

### Внески коду

Ми любимо внески коду! Ось як почати:

1. **Форкніть репозиторій** і створіть свою гілку з `master`
2. **Налаштуйте своє середовище розробки** (див. Налаштування розробки нижче)
3. **Внесіть свої зміни**, дотримуючись наших стандартів кодування
4. **Ретельно протестуйте свої зміни**
5. **Оновіть документацію**, якщо потрібно
6. **Надішліть pull request**, використовуючи наш шаблон PR

### Документація

Покращення документації завжди цінуються! Це включає:
- Файли README
- Сторінки Wiki
- Коментарі до коду
- Навчальні посібники та керівництва
- Переклади на інші мови

## Налаштування розробки

### Передумови

- Python 3.8 або вище
- Git
- Права Root/Адміністратора (для захоплення пакетів)

### Налаштування Вашого Середовища

```bash
# Клонуйте свій форк
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# Створіть віртуальне середовище
python3 -m venv venv

# Активуйте віртуальне середовище
# На Linux/macOS:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Встановіть залежності
pip install -r requirements.txt

# Встановіть залежності для розробки
pip install pylint flake8 black pytest
```

### Запуск Додатка

```bash
# На Linux/macOS:
sudo venv/bin/python3 albion_insight.py

# На Windows (як Адміністратор):
python albion_insight.py
```

## Стандарти кодування

Ми дотримуємося рекомендацій стилю PEP 8 для коду Python. Будь ласка, переконайтеся, що ваш код відповідає цим стандартам:

- Використовуйте 4 пробіли для відступу (без табуляції)
- Максимальна довжина рядка 100 символів
- Використовуйте змістовні імена змінних і функцій
- Додайте docstrings до всіх функцій і класів
- Включайте підказки типів (type hints) там, де це доречно
- Зберігайте функції сфокусованими та лаконічними

**Інструменти для допомоги:**
```bash
# Форматуйте свій код за допомогою black
black albion_insight.py

# Перевірте наявність проблем зі стилем
flake8 albion_insight.py

# Запустіть linter
pylint albion_insight.py
```

## Повідомлення комітів

Пишіть чіткі та змістовні повідомлення комітів:

- Використовуйте теперішній час ("Додати функцію", а не "Додав функцію")
- Використовуйте наказовий спосіб ("Перемістити курсор до...", а не "Переміщує курсор до...")
- Обмежте перший рядок до 72 символів
- Посилайтеся на проблеми та pull request-и, коли це доречно

**Приклади:**
```
Додати функціональність експорту лічильника шкоди
	
Виправити розбір мережевих пакетів для з'єднань IPv6
	
Оновити README інструкціями з інсталяції macOS
	
Закриває #123
```

## Процес Pull Request

1. **Оновіть документацію** для будь-яких змін у функціональності
2. **Додайте тести** для нових функцій або виправлень помилок
3. **Переконайтеся, що всі тести проходять** перед надсиланням
4. **Оновіть README.md**, якщо потрібно
5. **Повністю заповніть шаблон PR**
6. **Посилайтеся на пов'язані проблеми** у своєму описі PR
7. **Запитайте перевірку** у супровідників
8. **Оперативно та професійно реагуйте на відгуки**

### Контрольний список PR

Перш ніж надіслати свій PR, переконайтеся, що:
- [ ] Код відповідає рекомендаціям стилю проєкту
- [ ] Самоперевірка завершена
- [ ] Додано коментарі до складних розділів коду
- [ ] Документація оновлена
- [ ] Не створено нових попереджень
- [ ] Тести додано та вони проходять
- [ ] Залежні зміни об'єднано
