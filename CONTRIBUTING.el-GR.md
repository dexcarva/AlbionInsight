## Κώδικας Δεοντολογίας

Αυτό το έργο και όλοι όσοι συμμετέχουν σε αυτό διέπονται από τον Κώδικα Δεοντολογίας μας. Με τη συμμετοχή σας, αναμένεται να τηρήσετε αυτόν τον κώδικα. Παρακαλούμε αναφέρετε απαράδεκτη συμπεριφορά στους συντηρητές του έργου.

## Πώς μπορώ να συμβάλω;

### Αναφορά Σφαλμάτων

Πριν δημιουργήσετε αναφορές σφαλμάτων, ελέγξτε τα υπάρχοντα ζητήματα για να αποφύγετε διπλότυπα. Όταν δημιουργείτε μια αναφορά σφάλματος, συμπεριλάβετε όσο το δυνατόν περισσότερες λεπτομέρειες χρησιμοποιώντας το πρότυπο αναφοράς σφάλματος.

**Οι καλές αναφορές σφαλμάτων περιλαμβάνουν:**
- Έναν σαφή και περιγραφικό τίτλο
- Ακριβή βήματα για την αναπαραγωγή του προβλήματος
- Αναμενόμενη έναντι πραγματικής συμπεριφοράς
- Στιγμιότυπα οθόνης, εάν είναι εφαρμόσιμα
- Λεπτομέρειες του περιβάλλοντός σας (OS, έκδοση Python, κ.λπ.)
- Σχετικά αρχεία καταγραφής ή μηνύματα σφάλματος

### Πρόταση Λειτουργιών

Οι προτάσεις λειτουργιών είναι ευπρόσδεκτες! Χρησιμοποιήστε το πρότυπο αιτήματος λειτουργίας και δώστε:
- Μια σαφή περιγραφή της λειτουργίας
- Το πρόβλημα που επιλύει
- Πιθανές προσεγγίσεις υλοποίησης
- Οποιεσδήποτε εναλλακτικές λύσεις έχετε εξετάσει

### Συμβολές Κώδικα

Αγαπάμε τις συμβολές κώδικα! Δείτε πώς να ξεκινήσετε:

1. **Κλωνοποιήστε το αποθετήριο** (Fork) και δημιουργήστε το δικό σας branch από το `master`
2. **Ρυθμίστε το περιβάλλον ανάπτυξής σας** (δείτε Ρύθμιση Ανάπτυξης παρακάτω)
3. **Κάντε τις αλλαγές σας** ακολουθώντας τα πρότυπα κωδικοποίησης
4. **Δοκιμάστε τις αλλαγές σας** διεξοδικά
5. **Ενημερώστε την τεκμηρίωση** εάν χρειάζεται
6. **Υποβάλετε ένα pull request** χρησιμοποιώντας το πρότυπο PR

### Τεκμηρίωση

Οι βελτιώσεις στην τεκμηρίωση είναι πάντα ευπρόσδεκτες! Αυτό περιλαμβάνει:
- Αρχεία README
- Σελίδες Wiki
- Σχόλια κώδικα
- Εκπαιδευτικά υλικά και οδηγοί
- Μεταφράσεις σε άλλες γλώσσες

## Ρύθμιση Ανάπτυξης

### Προαπαιτούμενα

- Python 3.8 ή νεότερη έκδοση
- Git
- Προνόμια Root/Διαχειριστή (για λήψη πακέτων)

### Ρύθμιση του Περιβάλλοντός σας

```bash
# Κλωνοποιήστε το fork σας
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# Δημιουργήστε ένα εικονικό περιβάλλον
python3 -m venv venv

# Ενεργοποιήστε το εικονικό περιβάλλον
# Σε Linux/macOS:
source venv/bin/activate
# Σε Windows:
venv\Scripts\activate

# Εγκαταστήστε εξαρτήσεις
pip install -r requirements.txt

# Εγκαταστήστε εξαρτήσεις ανάπτυξης
pip install pylint flake8 black pytest
```

### Εκτέλεση της Εφαρμογής

```bash
# Σε Linux/macOS:
sudo venv/bin/python3 albion_insight.py

# Σε Windows (ως Διαχειριστής):
python albion_insight.py
```

## Πρότυπα Κωδικοποίησης

Ακολουθούμε τις οδηγίες στυλ PEP 8 για τον κώδικα Python. Βεβαιωθείτε ότι ο κώδικάς σας συμμορφώνεται με αυτά τα πρότυπα:

- Χρησιμοποιήστε 4 κενά για εσοχή (όχι tabs)
- Μέγιστο μήκος γραμμής 100 χαρακτήρες
- Χρησιμοποιήστε ουσιαστικά ονόματα μεταβλητών και συναρτήσεων
- Προσθέστε docstrings σε όλες τις συναρτήσεις και κλάσεις
- Συμπεριλάβετε υποδείξεις τύπου (type hints) όπου είναι σκόπιμο
- Διατηρήστε τις συναρτήσεις εστιασμένες και συνοπτικές

**Εργαλεία για βοήθεια:**
```bash
# Διαμορφώστε τον κώδικά σας με το black
black albion_insight.py

# Ελέγξτε για προβλήματα στυλ
flake8 albion_insight.py

# Εκτελέστε το linter
pylint albion_insight.py
```

## Μηνύματα Commit

Γράψτε σαφή και ουσιαστικά μηνύματα commit:

- Χρησιμοποιήστε τον ενεστώτα ("Προσθήκη λειτουργίας" όχι "Πρόσθεσε λειτουργία")
- Χρησιμοποιήστε την προστακτική διάθεση ("Μετακίνηση δρομέα σε..." όχι "Μετακινεί δρομέα σε...")
- Περιορίστε την πρώτη γραμμή σε 72 χαρακτήρες
- Αναφέρετε ζητήματα και pull requests όταν είναι σχετικά

**Παραδείγματα:**
```
Προσθήκη λειτουργικότητας εξαγωγής μετρητή ζημιάς
	
Διόρθωση ανάλυσης πακέτων δικτύου για συνδέσεις IPv6
	
Ενημέρωση README με οδηγίες εγκατάστασης για macOS
	
Κλείνει το #123
```

## Διαδικασία Pull Request

1. **Ενημερώστε την τεκμηρίωση** για τυχόν αλλαγές στη λειτουργικότητα
2. **Προσθέστε δοκιμές** για νέες λειτουργίες ή διορθώσεις σφαλμάτων
3. **Βεβαιωθείτε ότι όλες οι δοκιμές περνούν** πριν την υποβολή
4. **Ενημερώστε το README.md** εάν χρειάζεται
5. **Συμπληρώστε πλήρως το πρότυπο PR**
6. **Συνδέστε σχετικά ζητήματα** στην περιγραφή του PR σας
7. **Ζητήστε έλεγχο** από τους συντηρητές
8. **Αντιμετωπίστε τα σχόλια** άμεσα και επαγγελματικά

### Λίστα Ελέγχου PR

Πριν υποβάλετε το PR σας, βεβαιωθείτε ότι:
- [ ] Ο κώδικας ακολουθεί τις οδηγίες στυλ του έργου
- [ ] Η αυτοαξιολόγηση ολοκληρώθηκε
- [ ] Σχόλια προστέθηκαν σε σύνθετα τμήματα κώδικα
- [ ] Η τεκμηρίωση ενημερώθηκε
- [ ] Δεν δημιουργήθηκαν νέες προειδοποιήσεις
- [ ] Οι δοκιμές προστέθηκαν και περνούν
- [ ] Οι εξαρτώμενες αλλαγές συγχωνεύτηκαν
