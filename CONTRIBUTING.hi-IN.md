## आचार संहिता

यह परियोजना और इसमें भाग लेने वाला हर कोई हमारी आचार संहिता द्वारा शासित होता है। भाग लेकर, आपसे इस संहिता को बनाए रखने की अपेक्षा की जाती है। कृपया अस्वीकार्य व्यवहार की रिपोर्ट परियोजना के अनुरक्षकों को करें।

## मैं कैसे योगदान कर सकता हूँ?

### बग रिपोर्ट करना

बग रिपोर्ट बनाने से पहले, कृपया डुप्लिकेट से बचने के लिए मौजूदा इश्यू की जाँच करें। जब आप एक बग रिपोर्ट बनाते हैं, तो बग रिपोर्ट टेम्पलेट का उपयोग करके यथासंभव अधिक विवरण शामिल करें।

**अच्छी बग रिपोर्ट में शामिल हैं:**
- एक स्पष्ट और वर्णनात्मक शीर्षक
- समस्या को पुन: उत्पन्न करने के लिए सटीक चरण
- अपेक्षित बनाम वास्तविक व्यवहार
- यदि लागू हो तो स्क्रीनशॉट
- आपके पर्यावरण का विवरण (OS, Python संस्करण, आदि)
- प्रासंगिक लॉग या त्रुटि संदेश

### सुविधाओं का सुझाव देना

सुविधाओं के सुझावों का स्वागत है! कृपया सुविधा अनुरोध टेम्पलेट का उपयोग करें और प्रदान करें:
- सुविधा का स्पष्ट विवरण
- वह समस्या जिसे यह हल करती है
- संभावित कार्यान्वयन दृष्टिकोण
- कोई भी विकल्प जिस पर आपने विचार किया है

### कोड योगदान

हमें कोड योगदान पसंद हैं! यहाँ बताया गया है कि कैसे शुरू करें:

1. **रिपॉजिटरी को फोर्क करें** और `master` से अपनी शाखा बनाएँ
2. **अपना विकास वातावरण सेट करें** (नीचे विकास सेटअप देखें)
3. हमारे कोडिंग मानकों का पालन करते हुए **अपने बदलाव करें**
4. **अपने बदलावों का पूरी तरह से परीक्षण करें**
5. यदि आवश्यक हो तो **दस्तावेज़ीकरण अपडेट करें**
6. हमारे PR टेम्पलेट का उपयोग करके **एक पुल रिक्वेस्ट सबमिट करें**

### दस्तावेज़ीकरण

दस्तावेज़ीकरण में सुधार का हमेशा स्वागत है! इसमें शामिल हैं:
- README फ़ाइलें
- विकी पृष्ठ
- कोड टिप्पणियाँ
- ट्यूटोरियल और गाइड
- अन्य भाषाओं में अनुवाद

## विकास सेटअप

### आवश्यक शर्तें

- पायथन 3.8 या उच्चतर
- गिट
- रूट/एडमिनिस्ट्रेटर विशेषाधिकार (पैकेट कैप्चर के लिए)

### अपना वातावरण सेट करना

```bash
# अपनी फोर्क को क्लोन करें
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# एक वर्चुअल वातावरण बनाएँ
python3 -m venv venv

# वर्चुअल वातावरण सक्रिय करें
# लिनक्स/मैकओएस पर:
source venv/bin/activate
# विंडोज पर:
venv\Scripts\activate

# निर्भरताएँ स्थापित करें
pip install -r requirements.txt

# विकास निर्भरताएँ स्थापित करें
pip install pylint flake8 black pytest
```

### एप्लिकेशन चलाना

```bash
# लिनक्स/मैकओएस पर:
sudo venv/bin/python3 albion_insight.py

# विंडोज पर (एडमिनिस्ट्रेटर के रूप में):
python albion_insight.py
```

## कोडिंग मानक

हम पायथन कोड के लिए PEP 8 स्टाइल दिशानिर्देशों का पालन करते हैं। कृपया सुनिश्चित करें कि आपका कोड इन मानकों का पालन करता है:

- इंडेंटेशन के लिए 4 स्पेस का उपयोग करें (कोई टैब नहीं)
- 100 वर्णों की अधिकतम पंक्ति लंबाई
- सार्थक चर और फ़ंक्शन नामों का उपयोग करें
- सभी फ़ंक्शन और कक्षाओं में डॉकस्ट्रिंग जोड़ें
- जहाँ उपयुक्त हो, टाइप हिंट्स शामिल करें
- फ़ंक्शंस को केंद्रित और संक्षिप्त रखें

**मदद के लिए उपकरण:**
```bash
# black के साथ अपने कोड को फॉर्मेट करें
black albion_insight.py

# स्टाइल समस्याओं की जाँच करें
flake8 albion_insight.py

# लिंटर चलाएँ
pylint albion_insight.py
```

## कमिट संदेश

स्पष्ट और सार्थक कमिट संदेश लिखें:

- वर्तमान काल का उपयोग करें ("सुविधा जोड़ें" न कि "सुविधा जोड़ी गई")
- अनिवार्य मनोदशा का उपयोग करें ("कर्सर को ले जाएँ..." न कि "कर्सर को ले जाता है...")
- पहली पंक्ति को 72 वर्णों तक सीमित रखें
- प्रासंगिक होने पर इश्यू और पुल रिक्वेस्ट का संदर्भ लें

**उदाहरण:**
```
Add damage meter export functionality

Fix network packet parsing for IPv6 connections

Update README with macOS installation instructions

Closes #123
```

## पुल रिक्वेस्ट प्रक्रिया

1. कार्यक्षमता में किसी भी बदलाव के लिए **दस्तावेज़ीकरण अपडेट करें**
2. नई सुविधाओं या बग फिक्स के लिए **परीक्षण जोड़ें**
3. सबमिट करने से पहले **सुनिश्चित करें कि सभी परीक्षण पास हों**
4. यदि आवश्यक हो तो **README.md अपडेट करें**
5. **PR टेम्पलेट को पूरी तरह से भरें**
6. अपने PR विवरण में **संबंधित इश्यू को लिंक करें**
7. अनुरक्षकों से **समीक्षा का अनुरोध करें**
8. **प्रतिक्रिया को तुरंत और पेशेवर रूप से संबोधित करें**

### PR चेकलिस्ट

अपना PR सबमिट करने से पहले, सुनिश्चित करें:
- [ ] कोड परियोजना के स्टाइल दिशानिर्देशों का पालन करता है
- [ ] स्व-समीक्षा पूरी हो गई है
- [ ] जटिल कोड अनुभागों में टिप्पणियाँ जोड़ी गई हैं
- [ ] दस्तावेज़ीकरण अपडेट किया गया है
- [ ] कोई नई चेतावनी उत्पन्न नहीं हुई है
- [ ] परीक्षण जोड़े गए हैं और पास हो रहे हैं
- [ ] आश्रित बदलाव मर्ज किए गए हैं
