## Ёс зүйн дүрэм

Энэхүү төсөл болон түүнд оролцогч бүх хүмүүс манай Ёс зүйн дүрмээр зохицуулагддаг. Оролцохдоо та энэ дүрмийг сахих ёстой. Хүлээн зөвшөөрөгдөхгүй үйлдлийг төслийн менежерүүдэд мэдэгдэнэ үү.

## Би хэрхэн хувь нэмэр оруулж болох вэ?

### Алдааны тайлан

Алдааны тайлан үүсгэхээс өмнө, давхардлаас зайлсхийхийн тулд одоо байгаа асуудлуудыг шалгана уу. Алдааны тайлан үүсгэхдээ алдааны тайлангийн загварыг ашиглан аль болох олон дэлгэрэнгүй мэдээллийг оруулна уу.

**Сайн алдааны тайланд дараахь зүйлс орно:**
- Тодорхой бөгөөд тайлбарласан гарчиг
- Асуудлыг давтах нарийн алхамууд
- Хүлээгдэж буй болон бодит үйлдэл
- Хэрэв боломжтой бол дэлгэцийн зураг
- Таны орчны дэлгэрэнгүй мэдээлэл (OS, Python хувилбар гэх мэт)
- Холбогдох лог эсвэл алдааны мессежүүд

### Онцлог шинж чанаруудыг санал болгох

Онцлог шинж чанаруудын саналууд тавтай морилно уу! Онцлог шинж чанарын хүсэлтийн загварыг ашиглан дараахь зүйлсийг өгнө үү:
- Онцлог шинж чанарын тодорхой тайлбар
- Энэ нь шийдвэрлэх асуудал
- Хэрэгжүүлэх боломжит арга замууд
- Таны авч үзсэн бусад хувилбарууд

### Кодын хувь нэмэр

Бид кодын хувь нэмрийг маш их таашаадаг! Эхлэх арга:

1. **Репозиторийг Fork хийх** ба `master`-аас өөрийн салбарыг үүсгэх
2. **Хөгжүүлэлтийн орчноо тохируулах** (доорх Хөгжүүлэлтийн тохиргоог үзнэ үү)
3. **Өөрчлөлтөө хийх** манай кодчилолын стандартуудыг дагаж
4. **Өөрчлөлтөө сайтар турших**
5. **Баримт бичгийг шинэчлэх** шаардлагатай бол
6. **Pull Request илгээх** манай PR загварыг ашиглан

### Баримт бичиг

Баримт бичгийн сайжруулалт үргэлж талархалтай байдаг! Үүнд:
- README файлууд
- Wiki хуудсууд
- Кодын тайлбарууд
- Зааварчилгаа ба гарын авлагууд
- Бусад хэл рүү орчуулга

## Хөгжүүлэлтийн тохиргоо

### Урьдчилсан нөхцөл

- Python 3.8 эсвэл түүнээс дээш
- Git
- Root/Администраторын эрх (пакет барихын тулд)

### Таны орчныг тохируулах

```bash
# Өөрийн fork-ийг татаж авах
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight

# Виртуал орчин үүсгэх
python3 -m venv venv

# Виртуал орчинг идэвхжүүлэх
# Linux/macOS дээр:
source venv/bin/activate
# Windows дээр:
venv\Scripts\activate

# Хамаарлуудыг суулгах
pip install -r requirements.txt

# Хөгжүүлэлтийн хамаарлуудыг суулгах
pip install pylint flake8 black pytest
```

### Програмыг ажиллуулах

```bash
# Linux/macOS дээр:
sudo venv/bin/python3 albion_insight.py

# Windows дээр (Администратороор):
python albion_insight.py
```

## Кодчилолын стандартууд

Бид Python кодын хувьд PEP 8 загварын удирдамжийг дагаж мөрддөг. Таны код дараахь стандартуудыг хангаж байгаа эсэхийг шалгана уу:

- 4 зайг догол мөрөнд ашиглах (табгүй)
- Хамгийн их мөрийн урт 100 тэмдэгт
- Утга учиртай хувьсагч болон функцийн нэрийг ашиглах
- Бүх функц, классуудад docstring нэмэх
- Шаардлагатай бол төрлийн зөвлөмжийг оруулах
- Функцуудыг төвлөрсөн, товч байлгах

**Туслах хэрэгслүүд:**
```bash
# Кодоо black-аар форматлах
black albion_insight.py

# Загварын асуудлуудыг шалгах
flake8 albion_insight.py

# Linter-ийг ажиллуулах
pylint albion_insight.py
```

## Коммит мессежүүд

Тодорхой бөгөөд утга учиртай коммит мессежүүдийг бичнэ үү:

- Одоо цагийг ашиглах ("Онцлог шинж чанар нэмэх" биш "Онцлог шинж чанар нэмсэн")
- Тушаах хэлбэрийг ашиглах ("Курсорыг ... руу зөөх" биш "Курсорыг ... руу зөөдөг")
- Эхний мөрийг 72 тэмдэгтээр хязгаарлах
- Холбогдох үед асуудлууд болон pull request-уудыг дурдах

**Жишээнүүд:**
```
Хохирлын тоолуурын экспортлох функцийг нэмэх
	
IPv6 холболтуудад зориулсан сүлжээний пакет задлалыг засах
	
macOS суулгах зааврыг README-д шинэчлэх
	
#123-ийг хаах
```

## Pull Request-ийн үйл явц

1. **Баримт бичгийг шинэчлэх** аливаа үйл ажиллагааны өөрчлөлтөд зориулж
2. **Тестүүдийг нэмэх** шинэ онцлог эсвэл алдаа засахад зориулж
3. **Илгээхээс өмнө бүх тестүүд амжилттай болсон эсэхийг шалгах**
4. **README.md-г шинэчлэх** шаардлагатай бол
5. **PR загварыг бүрэн бөглөх**
6. **Холбогдох асуудлуудыг холбох** PR-ийн тайлбартаа
7. **Хянаж үзэхийг хүсэх** менежерүүдээс
8. **Санал хүсэлтийг хурдан бөгөөд мэргэжлийн түвшинд шийдвэрлэх**

### PR-ийн шалгах хуудас

PR-ээ илгээхээс өмнө дараахь зүйлсийг шалгана уу:
- [ ] Код нь төслийн загварын удирдамжийг дагаж мөрддөг
- [ ] Өөрийгөө хянах ажил дууссан
- [ ] Нарийн төвөгтэй кодын хэсгүүдэд тайлбар нэмсэн
- [ ] Баримт бичиг шинэчлэгдсэн
- [ ] Шинэ анхааруулга үүсээгүй
- [ ] Тестүүд нэмэгдсэн ба амжилттай болсон
- [ ] Хамааралтай өөрчлөлтүүд нэгтгэгдсэн
