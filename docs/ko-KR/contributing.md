## 행동 강령
이 프로젝트와 참여하는 모든 사람은 우리의 행동 강령을 따릅니다. 참여함으로써 여러분은 이 강령을 준수할 것으로 예상됩니다. 용납할 수 없는 행동은 프로젝트 관리자에게 보고해 주십시오.

## 어떻게 기여할 수 있나요?
### 버그 보고
버그 보고서를 작성하기 전에, 중복을 피하기 위해 기존 이슈를 확인하십시오. 버그 보고서를 작성할 때는 버그 보고서 템플릿을 사용하여 가능한 한 많은 세부 정보를 포함하십시오.
**좋은 버그 보고서에는 다음이 포함됩니다:**
- 명확하고 설명적인 제목
- 문제를 재현하기 위한 정확한 단계
- 예상되는 동작 대 실제 동작
- 해당되는 경우 스크린샷
- 환경 세부 정보 (OS, Python 버전 등)
- 관련 로그 또는 오류 메시지

### 기능 제안
기능 제안을 환영합니다! 기능 요청 템플릿을 사용하고 다음을 제공하십시오:
- 기능에 대한 명확한 설명
- 해결하는 문제
- 가능한 구현 접근 방식
- 고려했던 대안

### 코드 기여
코드 기여를 환영합니다! 시작하는 방법은 다음과 같습니다:
1. **저장소를 포크**하고 \`master\`에서 브랜치를 생성하십시오.
2. **개발 환경을 설정**하십시오 (아래 개발 환경 설정 참조).
3. 우리의 코딩 표준을 따르면서 **변경 사항을 만드십시오**.
4. **변경 사항을 철저히 테스트**하십시오.
5. 필요한 경우 **문서를 업데이트**하십시오.
6. 우리의 PR 템플릿을 사용하여 **Pull Request를 제출**하십시오.

### 문서화
문서 개선은 항상 환영합니다! 다음을 포함합니다:
- README 파일
- Wiki 페이지
- 코드 주석
- 튜토리얼 및 가이드
- 다른 언어로의 번역

## 개발 환경 설정
### 필수 요구 사항
- Python 3.8 이상
- Git
- Root/관리자 권한 (패킷 캡처용)

### 환경 설정
\`\`\`bash
# 포크 복제
git clone https://github.com/YOUR_USERNAME/AlbionInsight.git
cd AlbionInsight
# 가상 환경 생성
python3 -m venv venv
# 가상 환경 활성화
# Linux/macOS에서:
source venv/bin/activate
# Windows에서:
venv\Scripts\activate
# 종속성 설치
pip install -r requirements.txt
# 개발 종속성 설치
pip install pylint flake8 black pytest
\`\`\`

### 애플리케이션 실행
\`\`\`bash
# Linux/macOS에서:
sudo venv/bin/python3 albion_insight.py
# Windows에서 (관리자 권한으로):
python albion_insight.py
\`\`\`

## 코딩 표준
우리는 Python 코드에 대해 PEP 8 스타일 가이드라인을 따릅니다. 코드가 다음 표준을 준수하는지 확인하십시오:
- 들여쓰기에 4개의 공백 사용 (탭 사용 금지)
- 최대 줄 길이 100자
- 의미 있는 변수 및 함수 이름 사용
- 모든 함수 및 클래스에 docstring 추가
- 적절한 곳에 타입 힌트 포함
- 함수를 집중적이고 간결하게 유지
**도움이 되는 도구:**
\`\`\`bash
# black으로 코드 포맷팅
black albion_insight.py
# 스타일 문제 확인
flake8 albion_insight.py
# 린터 실행
pylint albion_insight.py
\`\`\`

## 커밋 메시지
명확하고 의미 있는 커밋 메시지를 작성하십시오:
- 현재 시제 사용 ("Add feature" 대신 "Added feature" 아님)
- 명령형 사용 ("Move cursor to..." 대신 "Moves cursor to..." 아님)
- 첫 줄을 72자로 제한
- 관련 이슈 및 Pull Request 참조
**예시:**
\`\`\`
Add damage meter export functionality
Fix network packet parsing for IPv6 connections
Update README with macOS installation instructions
Closes #123
\`\`\`

## Pull Request 프로세스
1. 기능 변경 사항에 대해 **문서를 업데이트**하십시오.
2. 새로운 기능 또는 버그 수정에 대해 **테스트를 추가**하십시오.
3. 제출하기 전에 **모든 테스트가 통과**하는지 확인하십시오.
4. 필요한 경우 **README.md를 업데이트**하십시오.
5. **PR 템플릿을 완전히 작성**하십시오.
6. PR 설명에 **관련 이슈를 연결**하십시오.
7. 관리자에게 **검토를 요청**하십시오.
8. **피드백에 신속하고 전문적으로 대응**하십시오.

### PR 체크리스트
PR을 제출하기 전에 다음을 확인하십시오:
- [ ] 코드가 프로젝트의 스타일 가이드라인을 따릅니다.
- [ ] 자체 검토 완료
- [ ] 복잡한 코드 섹션에 주석 추가
- [ ] 문서 업데이트
- [ ] 새로운 경고가 생성되지 않음
- [ ] 테스트가 추가되었고 통과함
- [ ] 종속된 변경 사항이 병합됨
